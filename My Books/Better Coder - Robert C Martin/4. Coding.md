
## Pacing Yourself 

* Rushing 
	"Do not tempted to Rush" 
	You will start taking shortcuts and working extra hours in hope of working miracle. This is a recipe of disaster because it sense of false hope to team and stake holders 
	
- False Delivery 
	False Done - we assume that the task is done and move on to next task. We give reason that it could be further completed when we have more time.
	Every one assume that things are on track but in reality it is a disaster. People will realize it when it is too late. 
	
- Define "Done" 
	let business analyst and tester write set of acceptance test that say when it is done.
