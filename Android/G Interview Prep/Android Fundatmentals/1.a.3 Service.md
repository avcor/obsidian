It is an android component, no UI 

### Types 
1. Foreground 
2. Background 
3. [[#Bound Service]] 

onStartCommand() return types 
1. START_NOT_STICKY
	If system kills the task then do not recreate it unless there is pending intents to deliver 
2. START_STICKY 
	If system kills the service after onStartCommand then recreate it but do not redeliver the Intent. eg for running the music player 
3. START_REDILEVER_INTENT
	Recreate the service with last dilevered intent, any pending intent are dilevered in turn.  eg download of a file 
### Bound Service
Allows android component to bind to service using bindService(). 
You can bound to a service which is already started through startService()

### Intent Service
IntentService extends Service depriciated instead use JobIntentSerive 
It is depriciated because limit on Background Execution Limits. 
It uses worker thread to handle all start request 
Process all request serially
